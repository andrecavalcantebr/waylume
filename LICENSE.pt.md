## Pontos principais

**Copyleft forte:** Qualquer trabalho derivado que você distribuir deve permanecer sob a GPLv3. Você não pode tornar o software proprietário.

**Código-fonte:** Ao distribuir o programa em formato binário/executável, você deve disponibilizar o código-fonte correspondente (ou uma oferta escrita válida por 3 anos).

**Patentes:** Se você redistribuir o programa, você concede automaticamente aos destinatários uma licença para as patentes que você controla e que sejam necessárias ao uso do programa.

**Tivoização proibida:** Se você distribuir o programa em hardware que impede o usuário de instalar versões modificadas (dispositivos de usuário final), isso é proibido pela GPLv3.

**Sem garantia:** O software é fornecido "como está", sem garantia de qualquer tipo — expressa ou implícita. O risco de qualidade e desempenho é inteiramente seu.

Sim. A GPLv3 permite uso comercial. Você pode vender o software — mas deve fornecer o código-fonte e manter a licença.

**Posso usar partes do código num projeto proprietário?**
Não. Qualquer trabalho derivado deve permanecer sob a GPLv3.

**Posso usar o software internamente sem redistribuir o código?**
Sim. A obrigação de disponibilizar o código-fonte só se aplica quando você *distribui* o software a terceiros.
